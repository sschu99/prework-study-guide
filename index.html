HTML

  • HTML elements give structure to the content of a webpage
  • The head element contains information about the webpage - metadata about accessibility, search engines and performance
  • The body element represents the visible content shown to the user - it is a HTML semantic element, where the name of the element describes its content/role
  • For visible HTML elements, content is contained between an opening and closing tag
  • HTML attributes define additional information about an element, defined in the opening element - eg. for the img element, src = source, alt = text that describes appearance/functionality of image
  • Elements nested inside other elements are called child elements
  • The footer normally contains information regarding the author, copyright, contact, sitemap, and navigation

CSS

  • There are three ways to style using CSS - inline CSS, internal CSS style sheet and external CSS style sheet
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates hwo much space we want around the content inside an element
  • * selector - rule applies to all elements visible on page
  • For selectors with the same style, combine them in a single CSS rule with a comma in between - avoid repeating codes
  • Link CCS file to HTML in the head element
  • Class attributes allow different elements in HTML to share a CSS rule, by assigning rule to a class attribute with a class selector
  • Adding font-family - font names with more than one word need to be surrounded by quotes

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds code in current working branch to staging area, which informs Git that these changes will be added to next commit
  • git commit -m "commit name": creates commit to repository
  • git pull origin main: to check if local branch is in sync with base branch in GitHub, pulls changes from base branch
  • git push origin branch-name: push changes to remote branch
  • In GitHub, compare and pull request - give a descriptive title and description of changes made
  • To merge the code from the feature branch into the main branch, must designate the base branch as main and comparison branch as current working branch.

JavaScript

  • A variable is a named container that allows us to store data in our code - declare a variable with let/var keyword, followed by the variable name
  • Control flow is the order in which a computer executes code in a script - in JS, code runs from top to bottom in a linear way
  • Conditional statements are used to test if expression returns true or not
  • Array is a structure that allows you to store multiple values in a single reference - can store strings, numbers, boolean value all in the same array - these values go within []
  • To access a stored data item in an array, you need to identify the index - arrays in JS are zero-indexed, in sequential whole numbers
  • The for loop can perform a task on all items in an array, where the single code block is executed repeatedly
  • The for loop contains 3 important statements:
  1. Determines the starting point for loop - where first index is always 0
  2. Condition - when no. in the variable x is less than length of array, loop will keep running
  3. Allows array to iterate over each item by writing x++, where the no. increases by 1 each time
  • Code to be executed in if statements, for loops and functions are placed within curly brackets { }
  • Functions are a set of instructions that tells the computer how to perform a task - creates a reusable block of code
  • Functions comprise of two requirements, firstly to define the function, and then call the function (to run/execute)
  • When linking JS file to HTML, ensure the code is at the end of the body element
